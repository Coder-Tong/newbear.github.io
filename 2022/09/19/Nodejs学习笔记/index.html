1 Node.js 简介

js 代码之所以能够运行在浏览器中，是因为浏览器提供了对应的运行时环境，也即是执行引擎。同样地，node.js 能够运行 js 代码，是由于：Nodejs 是基于 Chrome V8 JavaScript 引擎构建的 JavaScript 运行时环境。

  • 浏览器是JS的前端运行环境；
  • Node.js是JS的后端运行环境；
  • Node.js 中无法调用 DOM 和 BOM 等浏览器内置 API。

4 模块化

模块化就是遵循固定的规则，把一个大文件拆分成独立并且相互依赖的多个小模块。

代码模块化的好处如下：

  1. 提高了代码的复用性；
  2. 提高了代码的可维护性；
  3. 可以实现按需加载。

4.1 Nodejs 中模块化的分类

Nodejs中的模块大致可以分为三类：

  • 内置模块：Nodejs官方提供的模块，比如：fs、path以及http等；
  • 自定义模块：用户创建的 js 文件都属于自定义模块；
  • 第三方模块：由第三方提供的模块，非官方内置模块，使用前需要先下载。

Node.js 遵循 CommonJS 模块化规范， CommonJS 规定了模块的特性和各个模块之间如何相互依赖。

CommonJS 规定：

  1. 每个模块内部，module 变量代表当前模块；
  2. module 变量是一个对象，它的 exports 属性（即 module.exports）是对外的接口；
  3. 加载某个模块，其实是加载该模块的 module.exports 属性。require() 方法用于加载模块。
